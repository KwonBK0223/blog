> **Paper**: Communication-Efficient Learning of Deep Networks from Decentralized Data — H. Brendan McMahan, Eider Moore, Daniel Ramage, Seth Hampson, Blaise Agüera y Arcas, *AISTATS 2017*. [arXiv:1602.05629](https://arxiv.org/abs/1602.05629)

## TL;DR

- 모바일 기기에는 모델 학습에 유용한 풍부한 데이터가 있지만, 프라이버시에 민감하고 양이 방대해 데이터 센터로 모으기 어렵다.
- 데이터를 각자의 기기에 남겨둔 채 **locally-computed update만 집계해 공유 모델을 학습**하는 접근을 제안하고, 이를 **Federated Learning**이라 명명했다.
- 핵심 알고리즘 **FedAvg**는 각 클라이언트의 로컬 SGD와 서버의 모델 평균화를 결합한 것으로, 동기식 SGD(FedSGD) 대비 필요한 통신 라운드를 **10~100배** 줄인다.
- Unbalanced·non-IID 데이터 분포에서도 강건함을 다섯 가지 모델 아키텍처와 네 가지 데이터셋으로 실증했다.

## Background

### 왜 Federated Learning인가

연합학습은 중앙 서버의 조율 아래 참여 기기들(clients)의 느슨한 연합(loose federation)이 학습 문제를 푸는 구조다. 각 클라이언트는 로컬 데이터셋만 사용해 업데이트를 계산하고, 서버는 전달받은 업데이트만 집계해 global model을 갱신한다.

연합학습에 이상적인 문제의 조건은 다음과 같다.

1. 모바일 기기의 실제 데이터로 학습하는 것이 데이터 센터의 대리(proxy) 데이터로 학습하는 것보다 분명한 이점이 있다.
2. 데이터가 프라이버시에 민감하거나 크기가 커서 데이터 센터로 전송하는 것이 바람직하지 않다.
3. 지도학습의 경우 사용자 상호작용에서 레이블을 자연스럽게 추론할 수 있다.

프라이버시 관점에서도 전송되는 정보는 특정 모델을 개선하기 위한 **최소한의 정보**이고, 업데이트 자체는 일시적이며, raw 학습 데이터보다 적은 정보를 담고, 출처 추적이 필요 없다는 이점이 있다(완전한 보장은 아니며, 후술).

### Federated Optimization의 특성

연합학습에 내재된 최적화 문제를 저자들은 **federated optimization**이라 부르며, 일반적인 분산 최적화와 다음 네 가지로 구별한다.

- **Non-IID**: 각 클라이언트의 데이터는 특정 사용자의 기기 사용에 기반하므로 전체 분포를 대표하지 않는다.
- **Unbalanced**: 사용자마다 서비스 사용량이 달라 로컬 데이터 양이 크게 다르다.
- **Massively distributed**: 클라이언트 수가 클라이언트당 평균 데이터 수보다 훨씬 많다.
- **Limited communication**: 모바일 기기는 자주 오프라인이고 연결이 느리거나 비싸다.

목적 함수는 일반적인 ERM을 클라이언트 단위로 재구성한 형태다.

$$f(\omega) = \sum_{k=1}^{K} \frac{n_k}{n} F_k(\omega), \qquad F_k(\omega) = \frac{1}{n_k} \sum_{i \in P_k} f_i(\omega)$$

여기서 $K$는 클라이언트 수, $P_k$는 클라이언트 $k$의 데이터 인덱스 집합, $n_k = |P_k|$다.

데이터 센터의 최적화는 **계산 비용**이 지배하지만, 연합 환경에서는 **통신 비용**이 지배한다. 따라서 목표가 뒤집힌다 — 통신 라운드를 줄이기 위해 추가 계산을 쓰는 것이다. 방법은 두 가지: 병렬성 증가(더 많은 클라이언트 참여)와 **클라이언트당 계산량 증가**.

## Method

### FedSGD에서 FedAvg로

기준선인 **FedSGD**는 각 클라이언트가 로컬 데이터 전체에 대한 평균 gradient를 계산해 서버로 보내고, 서버가 이를 가중 집계해 한 걸음 업데이트하는 방식이다.

$$\omega_{t+1} \leftarrow \omega_t - \eta \sum_{k=1}^{K} \frac{n_k}{n} g_k, \qquad g_k = \nabla F_k(\omega_t)$$

라운드당 계산은 적지만 업데이트가 잦아 통신이 많이 필요하다.

**FedAvg**는 평균화하기 전에 각 클라이언트가 로컬 업데이트를 여러 번 반복한다. 알고리즘은 세 파라미터로 제어된다.

- $C$: 각 라운드에 참여하는 클라이언트 비율
- $E$: 각 클라이언트가 라운드마다 로컬 데이터로 수행하는 학습 epoch 수
- $B$: 로컬 미니배치 크기

서버 루프는 다음과 같다. 매 라운드 $t$마다 $m = \max(C \cdot K, 1)$개의 클라이언트를 무작위 선택하고, 각 클라이언트 $k$는 현재 global 파라미터 $\omega_t$에서 출발해 로컬 데이터를 배치 크기 $B$로 나눠 $E$ epoch 동안 $\omega \leftarrow \omega - \eta \nabla \ell(\omega; b)$를 수행한 뒤 $\omega_{t+1}^k$를 서버에 보낸다. 서버는 데이터 수 비례 가중 평균으로 갱신한다.

$$\omega_{t+1} \leftarrow \sum_{k \in S_t} \frac{n_k}{m_t}\, \omega_{t+1}^k, \qquad m_t = \sum_{k \in S_t} n_k$$

$B = \infty$(전체 배치), $E = 1$이면 FedSGD와 같아진다. 모델 파라미터를 평균하는 것이 non-convex 문제에서 잘 동작할 이유는 자명하지 않지만, 같은 초기값에서 출발한 모델들의 평균은 실험적으로 잘 동작함을 보였다.

## Experiments

**MNIST (2NN, CNN)** — IID/non-IID(숫자 클래스별 shard 분배) 분할 모두에서 실험했다. 목표 정확도(97%/99%)에 도달하는 데 필요한 통신 라운드를 세면, CNN 기준 FedAvg($C=0.1$, $E=5$, $B=10$)는 IID에서 18라운드로 FedSGD(387라운드) 대비 약 2.8배, non-IID에서도 4.6배 이상 빨랐다. 클라이언트 비율 $C$를 높이고 로컬 계산($E$↑, $B$↓)을 늘릴수록 수렴이 빨라진다.

**Shakespeare (LSTM, 문자 단위 언어 모델)** — 등장인물별로 데이터를 나눈 자연스러운 non-IID 설정에서, FedAvg($E=5$, $B=10$)는 54% 정확도까지 FedSGD 대비 IID에서 13.0배, **non-IID에서 95.3배** 적은 라운드로 도달했다. non-IID 데이터에서 FedAvg의 robustness가 오히려 두드러진다.

**CIFAR-10 (CNN)** — 목표 정확도 80% 기준 SGD 18,000라운드, FedSGD 3,750라운드, FedAvg는 **280라운드**(64.3배 절감). 85% 목표에서도 FedAvg만 2,000라운드로 도달했다.

정리하면 결론은 네 가지다.

- **Robust performance**: 실제 데이터에서 흔한 non-IID·unbalanced 분포에서도 강력하다.
- **Communication efficiency**: 수렴에 필요한 통신 라운드를 크게 줄여 연합 환경의 핵심 제약을 해결했다.
- **Accuracy**: 여러 데이터셋에서 FedSGD보다 높은 정확도(더 나은 일반화)를 달성했다.
- **Scalability**: 다양한 네트워크 크기와 데이터 분포에 대해 확장 가능하다.

## Takeaways

FedAvg는 "로컬에서 더 많이 계산하고 덜 자주 통신한다"는 단순한 아이디어로 연합학습을 실용 가능한 수준으로 끌어올렸고, 이후 등장한 수많은 FL 알고리즘(FedProx, SCAFFOLD 등)의 출발점이자 표준 baseline이 되었다.

다만 한계도 분명하다. 이후 연구인 Deep Leakage from Gradients (Zhu et al., 2019)에 따르면 공유되는 gradient로부터 특정 속성을 가진 샘플이 배치에 있는지 확인하거나, 생성 모델을 학습해 유사 데이터를 만들거나, 심지어 훈련 데이터를 복원하는 것까지 가능하다. 즉 **업데이트만 공유해도 완전한 프라이버시 보호는 불가능**하다. 이를 보완하기 위한 방향으로 다음이 연구되고 있다.

1. **Differential Privacy (DP)**: 노이즈를 추가해 개인 데이터를 구별할 수 없게 하는 방법
2. **Secure Multi-party Computation (SMPC)**: 입력을 비공개로 유지하며 함수를 공동 계산
3. **Homomorphic Encryption (HE)**: 암호화된 데이터에 대한 직접 계산
4. **Model Pruning / Quantization**: 모델 크기·복잡성을 줄여 출력에서의 데이터 재구성 위험 완화

연합학습이라는 분야 자체를 연 논문이라는 점에서, FL을 공부한다면 가장 먼저 읽어야 할 고전이다.
