> **Paper**: On Bridging Generic and Personalized Federated Learning for Image Classification — Hong-You Chen, Wei-Lun Chao, *ICLR 2022*. [arXiv:2107.00778](https://arxiv.org/abs/2107.00778)

## TL;DR

- 연합학습에는 두 가지 서로 다른 목표가 있다. **Generic FL(G-FL)** 은 하나의 global model을 잘 만드는 것이고, **Personalized FL(P-FL)** 은 각 client의 test data에 잘 맞는 client별 모델을 만드는 것이다.
- client마다 class distribution이 다른 non-IID 상황에서 이 두 목표는 충돌한다. **empirical risk**는 personalized 성능에 좋지만 client마다 최적화 목표가 달라져 global model을 해치고, **balanced risk**는 global model에 좋지만 personalized 성능을 떨어뜨릴 수 있다.
- **FedRoD**(Federated Robust Decoupling)는 shared feature extractor 위에 두 개의 predictor를 둔다. **G-head**는 balanced risk로 학습해 서버에서 집계하고, **P-head**는 empirical risk로 학습해 client local에 유지한다. personalized 예측은 두 head의 logit 합으로 정의된다.
- 그 결과 하나의 프레임워크로 G-FL과 P-FL 양쪽에서 동시에 좋은 성능을 얻으며, hypernetwork 확장을 통해 새로운 client에 대한 zero-shot adaptation도 지원한다.

## Background

### Generic FL vs Personalized FL

기존 연합학습 연구는 대부분 둘 중 하나의 목표에 집중해 왔다.

- **Generic FL**: 서버에 모이는 하나의 global model이 일반적인 test data에서 잘 동작하도록 만드는 것이 목적이다. FedAvg 계열 연구가 여기에 해당한다.
- **Personalized FL**: client마다 데이터 분포가 다르다는 이질성을 인정하고, 각 client $m$이 자기 local test data에 잘 맞는 전용 모델 $w_m$을 갖도록 하는 것이 목적이다.

일반적으로 P-FL 방법의 personalized model이 G-FL의 global model보다 각 client의 test data에서는 더 좋은 성능을 보인다. 문제는 두 목표를 동시에 잘하는 방법이 드물었다는 것이다.

### 용어 정리 — global/local vs generic/personalized

이 논문은 혼용되기 쉬운 용어를 명확히 구분하는데, 이 구분이 논문 전체를 읽는 데 중요하다.

- **Global model vs local model**은 **훈련 단계**의 구분이다.
  - **local model**: 각 라운드에서 client가 local 학습을 마친 직후의 모델.
  - **global model**: local model들이 서버에서 집계된 모델.
- **Generic model vs personalized model**은 **테스트 단계**의 구분이다.
  - **generic model**: 일반적인 future test data에 사용하는 모델.
  - **personalized model**: 각 client의 test data에 사용하는 모델.

즉 "local model"은 훈련 과정의 산물이고, "personalized model"은 테스트에서의 역할이다. 뒤에서 보겠지만, G-FL 알고리즘의 local model이 사실 꽤 강력한 personalized model이라는 관찰이 이 논문의 핵심 발견 중 하나다.

### non-IID의 두 형태

classification 관점에서 non-IID는 두 형태로 나눌 수 있다.

- **non-identical class distribution**: client마다 보유한 class의 비율 $P_m(y)$가 다른 경우.
- **non-identical class-conditional distribution**: 같은 class라도 feature의 분포 $P_m(x \mid y)$가 client마다 다른 경우.

후자는 client의 데이터를 직접 보지 않고는 완화하기 어렵다. 반면 전자, 즉 class distribution의 차이는 **client 간에 일관된 학습 목표를 설정하는 것**으로 상당 부분 완화할 수 있다는 것이 이 논문의 논지다. 구체적으로, class-balanced objective를 쓰면 client 간 local 훈련이 일관되어지고, 그 결과 global model이 non-IID 조건에 강건해진다.

### empirical risk와 balanced risk의 긴장 관계

**Empirical risk**는 client가 실제로 가진 local data에 대한 평균 loss다. client $m$의 데이터 분포를 그대로 따르므로, 이를 최소화하면 그 client의 test data에 잘 맞는 **personalized model에는 좋다**. 하지만 client마다 class distribution이 다르면 각 client가 서로 다른 목표를 최적화하게 되어, 이를 평균낸 **global model에는 나쁠 수 있다**.

**Balanced risk**는 class imbalance를 보정한 loss다. 일반 empirical risk는 client가 많이 가진 class 쪽으로 끌린다. 예를 들어 어떤 client가 class 0 데이터를 많이 갖고 있으면 모델이 class 0 중심으로 학습된다. balanced risk는 re-weighting이나 balanced softmax loss를 통해 각 class를 균형 있게 보도록 만들어, client마다 class 비율이 달라도 모든 client가 "모든 class를 잘 맞추자"는 **일관된 목표**를 갖게 한다. 이는 global model 성능에는 좋지만, client의 실제 local distribution을 그대로 맞추는 **personalized 성능은 오히려 떨어뜨릴 수 있다**. local model이 더 이상 client의 empirical risk를 최적화하지 않기 때문이다.

요약하면, empirical risk는 P-FL에, balanced risk는 G-FL에 유리하고 서로의 목표를 해친다. FedRoD는 이 긴장을 두 loss를 각각 다른 predictor에 할당하는 방식으로 해소한다.

## Method (FedRoD)

### 구조 — Two Losses, Two Predictors

FedRoD는 모델을 shared feature extractor와 두 개의 head로 분해한다.

$$z = f(x; \theta)$$

- $f(x;\theta)$: 모든 client가 공유하는 **feature extractor**.

$$\hat{y}_G = h^G(z; \psi)$$

- **G-head (generic predictor)**: global model을 위한 head로, **balanced risk**로 학습한다.

$$\hat{y}_P = h^G(z; \psi) + h^P(z; \phi_m)$$

- **P-head (personalized predictor)**: client $m$ 전용 head $h^P(z;\phi_m)$로, **empirical risk**로 학습한다. 독립된 예측기가 아니라 G-head 위에 **logit-level add-on**처럼 붙는다는 점이 핵심이다. personalized 예측은 "공통 예측 + client별 보정"으로 구성되며, 논문도 personalized prediction을 G-head와 P-head의 logit 합으로 정의한다.

이렇게 하면 feature extractor와 G-head는 client 간 일관된 balanced objective로 학습되어 집계에 안전하고, P-head는 각 client의 empirical distribution을 자유롭게 맞추면서도 G-head가 이미 잡아놓은 예측을 기반으로 작은 보정만 학습하면 된다.

### 학습 흐름

전체 파이프라인은 FedAvg를 따르며, 다음과 같이 진행된다.

1. Server가 shared feature extractor $\theta$와 G-head $\psi$를 client들에게 보낸다.
2. 각 client는 local data로 두 가지 loss를 동시에 학습한다.
3. $\theta, \psi$는 **balanced risk**로 학습한다.
4. personalized head $\phi_m$은 **empirical risk**로 학습한다.
5. client는 $\theta_m, \psi_m$만 server로 전송한다.
6. server는 $\theta, \psi$만 FedAvg처럼 평균낸다.
7. $\phi_m$은 client local에 유지된다.
8. test 시 generic한 데이터에는 $\theta, \psi$를 사용한다.
9. client-specific한 데이터에는 $\theta, \psi, \phi_m$을 사용한다.

즉 통신과 집계의 대상은 generic 부분($\theta, \psi$)뿐이고, personalized 부분($\phi_m$)은 각 client에 남는다. P-head가 경량 add-on이므로 추가 비용도 작다.

### Hypernetwork 확장 — FedRoD-hyper

FedRoD-hyper는 $\phi_m$을 client마다 따로 저장하는 대신, client의 class distribution $a_m$을 입력으로 받아 personalized head를 **생성**한다.

$$\phi_m = H^P(a_m; \nu)$$

$H^P$는 client의 class 분포를 넣으면 그 client에 맞는 personalized predictor를 만들어주는 작은 공유 hypernetwork로, 파라미터 $\nu$는 client들이 함께 학습한다. 장점은 새로운 client가 들어와도 class distribution만 알면 personalized head를 바로 만들 수 있다는 것이다. 논문은 이를 두 가지로 활용한다.

- **Zero-shot adaptation**: 새 client의 데이터 없이 class distribution만으로 personalized model을 즉시 제공한다.
- **Fine-tuning initialization**: 새 client가 자기 데이터로 fine-tuning할 때 더 좋은 시작점을 제공한다.

## Experiments

이 논문의 실험 결과는 세 가지 주장으로 요약된다.

- **G-FL과 P-FL 동시 달성**: 기존 방법들이 둘 중 하나에 특화된 것과 달리, FedRoD는 generic 성능과 personalized 성능 양쪽에서 동시에 강한 결과를 보인다. G-head 하나로는 강건한 global model을, G-head + P-head 조합으로는 각 client에 맞는 personalized model을 얻는다.
- **G-FL의 local model은 이미 강력한 personalized model**: 흥미로운 관찰로, FedAvg 같은 generic FL 알고리즘의 local 훈련 단계에서 나오는 local model이 이미 상당히 강력한 personalized model이라는 점을 보인다. 매 라운드 가중치 평균으로 만들어진 global model에서 local 학습을 시작하는 것이 **implicit regularization**으로 작용하기 때문이다. 즉 가중치 평균 initialization 자체가 강한 regularization이다.
- **class-balanced objective의 효과**: client들이 서로 다른 class distribution을 가질 때, class-balanced objective로 local 훈련을 일관되게 만드는 것이 generic FL 성능을 향상시킨다는 것을 실험적으로 확인한다.

또한 FedRoD-hyper는 학습에 참여하지 않은 새로운 client에 대해 zero-shot adaptation과 효과적인 fine-tuning 초기화를 제공한다.

## Takeaways

FedRoD는 "generic과 personalized는 다른 목표이고, 따라서 다른 loss와 다른 predictor가 필요하다"는 단순한 원칙을 구조로 옮긴 방법이다. 정리하면 다음과 같다.

- **역할의 명시적 분리**: global model이 잘하는 것(모든 class를 균형 있게)과 personalized model이 잘하는 것(내 분포를 그대로)은 다르며, 이를 하나의 모델·하나의 loss로 뭉뚱그리는 것이 기존 방법들의 한계였다. FedRoD는 balanced risk의 G-head와 empirical risk의 P-head로 역할을 분리하되, P-head를 logit add-on으로 설계해 두 역할이 표현을 공유하게 만들었다.
- **집계 대상의 선택**: FedBN이 "BN은 평균내지 않는다"로 feature shift에 대응했듯, FedRoD는 "P-head는 평균내지 않는다"로 class distribution shift에 대응한다. 무엇을 공유하고 무엇을 local에 남길지 선택하는 것이 personalized FL 설계의 핵심 축임을 다시 보여준다.
- **local model의 재해석**: G-FL의 local model이 implicit regularization 덕분에 이미 좋은 personalized model이라는 관찰은, G-FL과 P-FL 문헌을 잇는 다리로서 이후 연구에서 자주 인용되는 통찰이다.
- **분포 기반 personalization**: class distribution을 입력으로 personalized head를 생성하는 hypernetwork 확장은, 데이터 없이 분포 정보만으로 새 client에 적응한다는 실용적인 방향을 제시한다.

class distribution 차이가 non-IID의 주된 원인인 상황이라면, FedRoD는 G-FL과 P-FL 어느 쪽도 포기하지 않는 간결한 baseline이 된다.
