> **Paper**: BHFVAL: Blockchain-Enabled Hierarchical Federated Variational Autoencoder Framework for Secure Intrusion Detection in Vehicular Networks — G. E. Visuvanathan, Md S. Sayeed, S. Yogarayan, *Scientific Reports*, vol. 15, art. 45742, Dec. 2025. [doi:10.1038/s41598-025-28560-y](https://doi.org/10.1038/s41598-025-28560-y)

## TL;DR

차량 네트워크(IoV) 침입탐지를 위해 계층형 연합 Variational Autoencoder(HFVAL)를 중심에 두고, 블록체인 기반 검증(RBFT), 경량 엣지 컴퓨팅(LEC), 메타휴리스틱 하이퍼파라미터 최적화(OOA), 경량 보안 통신 프로토콜(LSCP)까지 결합한 종합 프레임워크다. Scalable, privacy-preserving, resource-efficient FL을 목표로 security, efficiency, flexibility를 동시에 강화하려는 시도다.

## Background

클라이언트가 많아지면 cloud server가 병목이 된다. HFL은 cloud–edge–client 구조를 통해 로컬 업데이트를 다중 edge가 먼저 받고, edge들이 cloud server에 파라미터를 전송하는 방식으로 이 문제를 완화한다. 이 구조는 더 많은 클라이언트가 프레임워크에 참여할 수 있게 하고 계산 부하를 절감한다.

이 논문의 시스템은 **edge node, fog-level RSU, cloud center**의 3계층으로 구성된다. Edge는 차량과 가장 가까운 계산 자원으로 차량 근처에서 계산·오프로딩·부분 aggregation을 수행하고, fog는 edge보다 상위의 중간 계층으로 여러 차량 또는 edge update를 모아 중간 aggregation을 수행한다.

또한 이 논문은 **zero-day attack** 대응을 강조한다. Zero-day attack이란 방어자 입장에서 아직 알려지지 않았거나, 학습 데이터에 충분히 포함되지 않았거나, 탐지 규칙이 준비되지 않은 새로운 공격을 말한다.

## Method

프레임워크의 주요 구성 요소는 다음과 같다.

- **RBFT(Reputation-based Byzantine Fault Tolerance) 블록체인**: 모델 집계 보호와 변조 방지 파라미터 저장을 보장한다. 블록체인은 raw 데이터를 저장하는 용도가 아니라, update 모델의 무결성을 보장하는 위변조가 어려운 공유 기록장부 역할이다.
- **경량 엣지 컴퓨팅(LEC)**: 차량 노드에서 인근 edge server로 계산을 동적으로 오프로드하여 지연 시간을 최소화한다.
- **Osprey Optimization Algorithm(OOA)**: 실시간 제약 조건 하에서 수렴성을 개선하기 위한 하이퍼파라미터 최적화에 사용된다.
- **LSCP 기반 ECHE**: 경량 보안 통신 프로토콜을 통해 최소한의 계산 오버헤드로 암호화된 V2X 통신을 가능하게 한다.
- **클래스 불균형 처리**: SMOTE 기반으로 majority class undersampling과 minority class oversampling을 결합한다. SMOTE는 "가짜 공격 데이터"를 무작위로 만드는 것이 아니라, 기존 minority class 샘플들의 feature space 안에서 새로운 minority-like 샘플을 생성하는 방식이다.

### 전체 학습 흐름

한 communication round의 흐름을 정리하면 다음과 같다.

1. **데이터 전처리**: 각 클라이언트의 로컬 데이터에서 불필요한 feature 제거, one-hot encoding, min-max scaling, 결측치 처리를 수행한다.
2. **초기 global model 준비**: 최상위 cloud server가 초기 모델 또는 이전 round의 global model을 준비한다.
3. **Cloud → fog/RSU/edge 전송**: Cloud는 모든 차량과 직접 통신하지 않고 중간 aggregator 계층으로 모델을 내려보낸다.
4. **Edge → 차량 배포**: 각 edge/RSU가 연결된 차량 노드들에게 global model을 배포한다.
5. **VAE encoding**: 차량은 raw data를 직접 공유하지 않고 VAE encoder를 통해 latent feature representation으로 변환한다.
6. **로컬 IDS model update**: 각 차량이 로컬 데이터 또는 latent representation으로 local gradient descent를 수행한다.
7. **OOA 하이퍼파라미터 조정**: learning rate, batch size, momentum 등을 동적으로 조정한다.
8. **LEC 연산 배치**: 차량의 계산 자원과 네트워크 상태에 따라 학습·집계 작업을 차량, edge, fog 중 적절한 계층에 분배한다.
9. **암호화 전송**: LSCP와 ECHE를 사용해 local update를 secure V2X communication으로 RSU/fog/edge에 전송한다.
10. **중간 aggregator의 update 수집**: 여러 차량 클라이언트로부터 local update를 모은다.
11. **RBFT 검증**: reputation score, signature, update consistency를 기반으로 악성 또는 품질이 낮은 update를 걸러낸다.
12. **Edge/fog-level aggregation**: 검증된 update만으로 reputation-weighted aggregation을 수행한다. 수식상으로는 FedAvg 계열의 weighted averaging에 가깝다.
13. **블록체인 기록**: model update hash, reputation score, signature 등을 블록체인 원장에 저장해 위변조를 방지한다.
14. **Fog/edge → cloud 전송**: 각 중간 aggregator가 집계한 model update를 cloud로 올린다.
15. **Cloud global aggregation**: 여러 edge/fog aggregated model을 다시 통합해 새로운 global model을 만든다.
16. **재배포**: cloud → fog/RSU/edge → vehicle 순으로 새 global model을 전달한다.
17. 다음 communication round를 반복한다.

요약하면, **vehicle local VAE 기반 representation/update → RSU/fog reputation-weighted aggregation → blockchain/RBFT 검증 → cloud global aggregation → 재배포**의 파이프라인이다.

## Experiments

- **데이터셋**: UNSW-NB15, CIC-IDS-2017
- 지연 시간 및 자원 활용 분석을 통해 LSCP/ECHE의 낮은 통신 오버헤드를 검증한다.

## Takeaways

- HFL + VAE + 블록체인 + 최적화 알고리즘 + 보안 통신을 한 프레임워크에 통합한, 구성 요소가 많은 시스템 논문이다.
- 집계 자체는 reputation-weighted FedAvg 계열이며, 블록체인은 update 무결성 검증 용도로 제한적으로 사용된다.
- 다만 몇 가지 불명확한 점이 있다. 로컬 학습 loss가 VAE loss인지 classification loss인지, 태스크가 classification인지 anomaly detection인지, 그리고 제안된 aggregation 수식이 FedAvg와 실질적으로 얼마나 다른지가 논문에서 명확히 서술되지 않는다.
