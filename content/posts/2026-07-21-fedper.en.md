> **Paper**: Federated Learning with Personalization Layers — Manoj Ghuhan Arivazhagan, Vinay Aggarwal, Aaditya Kumar Singh, Sunav Choudhary, 2019. [arXiv:1912.00818](https://arxiv.org/abs/1912.00818)

## TL;DR

- FedAvg assumes that all clients share a single global model, but on personalization tasks each client has different preferences and labeling criteria, leading to strong statistical heterogeneity.
- FedPer decomposes a deep model into **shared base layers** and **client-specific personalization layers**. Only the base layers are aggregated via FedAvg, while the personalization layers are trained on each client's local data and never sent to the server.
- By jointly learning a shared representation while keeping a client-specific head local, FedPer achieves faster convergence and higher accuracy than FedAvg in non-IID settings.

## Background

FedAvg assumes that every participating client learns the same global model. Each client computes a local update, the server aggregates these updates by a weighted average, and the process converges to a single model — so ultimately all clients share the same parameters.

This assumption breaks down on personalization tasks, because clients differ in both preferences and labeling criteria. On subjective problems such as image aesthetics assessment, for instance, **the same image may be assigned different labels by different users**. In such cases the data distribution is severely non-IID across clients, and because the labels themselves depend on the user, the statistical heterogeneity is especially pronounced.

Under these conditions, a single global model struggles to capture client-specific preferences. Forcing convergence to one parameter set that satisfies all clients can actually degrade personalization performance for each of them. FedPer addresses this by splitting the model architecture itself.

## Method

### Base layers and personalization layers

The central idea of FedPer is to decompose a deep model into two parts. The model for client $j$ is written as:

$$f(x; W_B, W_{P,j})$$

- $W_B$: the **base layers** shared by all clients. These are the layers closer to the input and are responsible for a general, shared representation.
- $W_{P,j}$: the **personalization layers** held only by client $j$. These are the layers closer to the output and act as a client-specific head that reflects that client's preferences and labeling criteria.

The base layers are aggregated on the server via FedAvg and shared across all clients, whereas the personalization layers are trained on each client's local data only and are never sent to the server. In other words, the only parameters exchanged with the server are the base layers.

For local optimization, the paper assumes SGD (or a variant thereof) as the local optimizer to keep the exposition simple.

### Training workflow

A single training round proceeds in the following five steps.

1. The server broadcasts the global base layers $W_B$ to the clients.
2. Each client trains the received base layers together with its own personalization layers on its local data.
3. Each client sends **only the updated base layers** back to the server.
4. The server aggregates only the received base layers by a data-size-weighted average to update the global base layers.
5. The personalization layers are not sent to the server and are retained within each client.

In this design, the base layers still enjoy the benefit of learning a common representation from many clients' data, while the personalization layers are never blended and canceled out by conflicting client preferences during aggregation. By keeping the personalization layers outside the FedAvg procedure, FedPer mitigates the adverse effect of statistical heterogeneity.

## Experiments

- **Datasets**: CIFAR-10, CIFAR-100, FLICKR-AES
- **Backbones**: ResNet-34, MobileNet-v1

On non-IID partitions of CIFAR, FedPer showed faster convergence and higher accuracy than FedAvg. The more the per-client data distributions diverge, the harder it becomes to train a single global model, and keeping the personalization layers local eases this difficulty.

On FLICKR-AES, an image aesthetics assessment dataset, differences in per-user preference are inherently large. In this setting FedAvg barely captured user preferences, whereas FedPer demonstrated a clear personalization effect through its client-specific head.

## Takeaways

FedPer, with its idea of separating a deep model into a **shared representation and client-specific head/layers**, is an early work that mitigates the limitations of FedAvg on personalization FL with strong statistical heterogeneity. It became a conceptual foundation for the personalization federated learning line of research that followed.

There are limitations as well. Deciding which layers form the base and where personalization begins is a design choice that must be made in advance, and the optimal split point varies with the model architecture and the task. Moreover, because the personalization layers are trained only on each client's local data, they risk overfitting on clients with little data. Even so, by raising the question of "what to share and what to personalize" at the level of model architecture, FedPer is a paper worth studying to understand personalization FL.
