> **Paper**: Federated Learning with Personalization Layers — Manoj Ghuhan Arivazhagan, Vinay Aggarwal, Aaditya Kumar Singh, Sunav Choudhary, 2019. [arXiv:1912.00818](https://arxiv.org/abs/1912.00818)

## TL;DR

- FedAvg는 모든 클라이언트가 하나의 global model을 공유한다고 가정하지만, personalization task에서는 클라이언트마다 선호와 label 기준이 달라 통계적 이질성(statistical heterogeneity)이 크다.
- FedPer는 딥러닝 모델을 **공유되는 base layers**와 **클라이언트별 personalization layers**로 분해한다. base layers만 FedAvg 방식으로 집계하고, personalization layers는 각 클라이언트가 로컬 데이터로만 학습하며 서버로 전송하지 않는다.
- shared representation을 함께 학습하면서도 client-specific head를 로컬에 유지함으로써, non-IID 환경에서 FedAvg보다 빠른 수렴과 높은 정확도를 보인다.

## Background

FedAvg는 참여하는 모든 클라이언트가 동일한 global model을 학습한다고 가정한다. 각 클라이언트가 로컬 업데이트를 계산하고 서버가 이를 가중 평균해 하나의 모델로 수렴시키는 구조이므로, 궁극적으로는 모든 클라이언트가 같은 파라미터를 공유하게 된다.

그러나 personalization task에서는 이 가정이 잘 맞지 않는다. 클라이언트마다 선호도와 label 기준이 다르기 때문이다. 예를 들어 이미지 미학 평가(aesthetics)처럼 주관이 개입하는 문제에서는 **같은 이미지라도 사용자마다 다른 label을 부여**할 수 있다. 이런 상황에서 데이터 분포는 클라이언트 간에 심하게 non-IID하며, 라벨 자체가 사용자에 종속된다는 점에서 통계적 이질성이 특히 크다.

이 경우 하나의 global model만으로는 client-specific preference를 반영하기 어렵다. 모든 클라이언트를 만족시키는 단일 파라미터로 수렴시키려는 시도가 오히려 각 클라이언트의 개인화 성능을 떨어뜨릴 수 있기 때문이다. FedPer는 이 문제를 모델 구조를 나누는 방식으로 접근한다.

## Method

### Base layers와 Personalization layers

FedPer의 핵심 아이디어는 딥러닝 모델을 두 부분으로 분해하는 것이다. 클라이언트 $j$의 모델을 다음과 같이 표기한다.

$$f(x; W_B, W_{P,j})$$

- $W_B$: 모든 클라이언트가 공유하는 **base layers**. 입력에 가까운 층으로, 일반적인 특징 표현(shared representation)을 담당한다.
- $W_{P,j}$: 클라이언트 $j$만 가지는 **personalization layers**. 출력에 가까운 층으로, 해당 클라이언트의 선호와 label 기준을 반영하는 client-specific head 역할을 한다.

base layers는 FedAvg 방식으로 서버에서 집계되어 모든 클라이언트가 공유하지만, personalization layers는 각 클라이언트가 로컬 데이터로만 학습하고 서버로 전송하지 않는다. 즉 서버와 주고받는 파라미터는 오직 base layers뿐이다.

로컬 최적화의 경우, 논문은 설명을 단순화하기 위해 SGD(또는 그 변형)를 로컬 옵티마이저로 가정한다.

### Training Workflow

한 라운드의 학습 절차는 다음 다섯 단계로 정리된다.

1. 서버가 global base layers $W_B$를 클라이언트들에게 broadcast한다.
2. 각 클라이언트는 전달받은 base layers와 자신의 personalization layers를 함께, 로컬 데이터로 학습한다.
3. 클라이언트는 업데이트된 **base layers만** 서버로 전송한다.
4. 서버는 전송받은 base layers만 데이터 수 비례 weighted averaging으로 집계해 global base layers를 갱신한다.
5. personalization layers는 서버로 전송되지 않고 각 클라이언트 내부에 유지된다.

이 구조에서 base layers는 여러 클라이언트의 데이터를 통해 공통 표현을 학습하는 이점을 그대로 누리는 한편, personalization layers는 집계 과정에서 서로 다른 클라이언트의 선호가 섞여 상쇄되는 일을 겪지 않는다. FedAvg 절차에서 벗어난 personalization layers가 통계적 이질성의 부정적 영향을 완화하는 셈이다.

## Experiments

- **Datasets**: CIFAR-10, CIFAR-100, FLICKR-AES
- **Backbones**: ResNet-34, MobileNet-v1

non-IID로 분할한 CIFAR 설정에서 FedPer는 FedAvg 대비 더 빠른 수렴과 더 높은 정확도를 보였다. 클라이언트별 데이터 분포가 서로 다를수록 단일 global model의 학습이 어려워지는데, personalization layers를 로컬에 두는 것이 이 어려움을 덜어준다.

이미지 미학 평가 데이터셋인 FLICKR-AES에서는 사용자별 선호 차이가 본질적으로 크다. 이 설정에서 FedAvg는 user preference를 거의 반영하지 못한 반면, FedPer는 client-specific head를 통해 personalization 효과를 확인할 수 있었다.

## Takeaways

FedPer는 deep model을 **shared representation과 client-specific head/layers로 분리**한다는 아이디어로, 통계적 이질성이 강한 personalization FL에서 FedAvg의 한계를 완화한 초기 연구다. 이후 등장하는 personalization federated learning 계열 연구들의 개념적 기반이 되었다.

다만 한계도 있다. 어떤 층까지를 base로 두고 어디부터를 personalization으로 둘지는 사전에 정해야 하는 설계 선택이며, 모델 구조와 태스크에 따라 최적 분할 지점이 달라진다. 또한 personalization layers가 각 클라이언트의 로컬 데이터로만 학습되므로, 데이터가 적은 클라이언트에서는 해당 층이 과적합될 위험이 있다. 그럼에도 "무엇을 공유하고 무엇을 개인화할 것인가"라는 질문을 모델 구조 차원에서 던졌다는 점에서, personalization FL을 이해하려면 짚고 넘어갈 논문이다.
