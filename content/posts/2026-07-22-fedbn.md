> **Paper**: FedBN: Federated Learning on Non-IID Features via Local Batch Normalization — Xiaoxiao Li, Meirui Jiang, Xiaofei Zhang, Michael Kamp, Qi Dou, *ICLR 2021*. [arXiv:2102.07623](https://arxiv.org/abs/2102.07623)

## TL;DR

- 기존 연합학습의 non-IID 연구는 주로 **label distribution skew**에 집중했지만, 현실에서는 client마다 입력 **feature 분포가 다른 feature shift**가 흔하게 발생한다.
- 이런 상황에서 모든 layer를 평균내는 FedAvg는 client별 feature 통계를 뭉개버려 성능이 떨어진다.
- **FedBN**은 FedAvg처럼 학습하되 **Batch Normalization(BN) layer만 평균내지 않고 client local에 유지**한다. 추가 hyperparameter가 없고 기존 FedAvg 구조에 그대로 얹을 수 있다.
- 단순화된 이론 분석과 digit benchmark, real-world multi-domain dataset 실험 모두에서 feature shift non-IID 설정에서 FedAvg·FedProx를 능가한다.

## Background

### non-IID의 두 종류

연합학습에서 non-IID는 하나가 아니라 서로 다른 유형으로 나뉜다. 기존 FL 연구가 주로 다룬 것은 **label distribution skew**로, client마다 보유한 class의 비율이 다른 경우다. 반면 이 논문이 초점을 두는 것은 **feature shift**, 즉 입력 feature의 분포 $P_i(x)$ 자체가 client마다 다른 상황이다.

feature shift는 현실에서 매우 자연스럽게 발생한다.

- **의료 영상**: 병원마다 MRI/CT 장비가 달라 영상의 intensity와 contrast가 다르다.
- **자율주행**: 차량마다 센서 특성, 주행 환경, 도로 상황이 다르다.
- **숫자 인식**: 같은 digit classification이라도 MNIST, SVHN, USPS처럼 이미지 스타일이 다르다.

### covariate shift와 concept shift

feature shift를 좀 더 세분하면 두 가지 개념으로 나눌 수 있다.

- **covariate shift**: 입력 $x$의 분포는 client마다 다르지만, $x$가 주어졌을 때 label $y$가 결정되는 방식 $P(y \mid x)$은 동일한 경우다. 예를 들어 병원마다 영상 contrast는 다르지만, 종양 여부를 판단하는 기준 자체는 같다.
- **concept shift**: label 비율은 비슷하지만 같은 class 안의 feature 모양이 client마다 다른 경우다. 모든 병원의 정상/질병 비율은 비슷하더라도, 질병 영상의 appearance가 장비마다 다르게 보이는 상황이 여기에 해당한다.

FedBN의 핵심 주장은, 이처럼 client별 feature distribution이 다를 때 BN 통계와 BN parameter까지 평균내는 것이 오히려 성능을 해칠 수 있다는 것이다.

## Method

### 핵심 아이디어 — BN을 local로 유지

FedBN은 FedAvg처럼 학습하되, **Batch Normalization layer만 평균내지 않고 client local에 남겨두는** 방법이다. Conv/FC layer는 FedAvg 방식으로 server에서 aggregation하고, BN layer는 각 client에 그대로 유지한다. 추가 hyperparameter가 없고, BN layer를 포함한 기존 FL 신경망에 쉽게 적용할 수 있다는 점이 장점이다.

워크플로는 다음과 같다.

1. Server가 global **non-BN parameter**를 client들에게 broadcast한다.
2. 각 client가 자기 local data와 **local BN**으로 전체 model을 학습한다.
3. 학습 후 client는 model parameter를 server로 업로드한다.
4. Server는 **non-BN layer만** 평균 낸다.
5. **BN layer는 client별 local parameter로 유지**된다.
6. 평균된 non-BN layer를 다시 client에게 배포하고, 각 client는 자기 BN으로 feature shift를 보정하면서 학습을 이어간다.

### 왜 local BN이 통하는가

Batch Normalization의 변환은 다음과 같이 쓸 수 있다.

$$y = \gamma \frac{x - \mu_B}{\sqrt{\sigma_B^2 + \epsilon}} + \beta$$

여기서 $\mu_B$와 $\sigma_B^2$는 batch로부터 추정되는 **client-specific feature 통계량**이고, $\gamma$와 $\beta$는 정규화된 feature를 다시 조절하는 **학습 가능한 scale/shift parameter**다. $\gamma$는 normalization 이후 feature의 크기를 다시 스케일링하는 값이라고 이해하면 된다.

client마다 feature distribution이 다르면, 이를 정규화하는 통계량 $\mu_B, \sigma_B^2$뿐 아니라 scale/shift parameter $\gamma, \beta$도 client마다 달라야 한다. 따라서 이들을 평균내는 대신 client-specific하게 두면, **local BN이 각 client의 feature shift를 흡수**하고 나머지 non-BN layer는 client가 공유하는 representation을 학습하게 된다.

## Theory & Experiments

### 이론 — feature shift를 가정하고 수렴을 분석

이론 파트에서는 feature shift를 **Assumption 4.1**로 수학적으로 모델링한다. 핵심은 각 client $i$의 input feature가 서로 다른 covariance를 가질 수 있다는 것이다. 평균을 0으로 두면 covariance는 다음과 같이 정의된다.

$$\mathbb{E}[x_i] = 0, \qquad S_i = \mathbb{E}[x_i x_i^\top]$$

여기서 $S_i$가 client마다 다를 수 있다. 즉 label distribution의 차이가 아니라 **feature distribution의 covariance 구조가 client마다 다른 상황**을 가정한다. 논문은 이를 "local feature distributions differ — not label distribution"이라고 명시한다.

이 가정 아래에서 FedAvg와 FedBN의 수렴률을 비교한다. 다만 분석은 일반적인 모든 deep network가 아니라, **two-layer ReLU network, squared loss, gradient descent, over-parameterized regime, local update $E=1$** 같은 단순화된 setting에서 진행된다. NTK/Gram matrix 관점에서 FedBN의 Gram matrix가 더 유리한 최소 eigenvalue를 가질 수 있음을 보이고, 결과적으로 FedBN이 FedAvg보다 빠른 convergence rate를 가질 수 있음을 주장한다.

### 실험 — feature shift setting을 직접 구성

실험도 단순히 feature shift가 있을 것이라고 말만 한 것이 아니라, client별 feature domain이 실제로 다르도록 구성했다. 대표 benchmark는 다섯 개의 digit dataset을 사용한다.

- SVHN
- USPS
- SynthDigits
- MNIST-M
- MNIST

이 다섯 dataset은 모두 digit classification이지만 이미지 appearance가 다르다. 즉 label space는 유사하되 input feature distribution이 다른, 전형적인 feature shift 상황이다. 논문은 각 dataset을 서로 다른 domain으로 보고 각 client가 하나의 domain만 갖도록 구성한다. 그래서 client 간 차이는 label skew라기보다 **image style, background, contrast, appearance 차이**에 가깝다. 또한 dataset별 client 수를 조절해 feature heterogeneity 수준을 바꿔가며 FedBN과 FedAvg를 비교한다.

benchmark뿐 아니라 실제 feature shift가 자연스럽게 발생하는 real-world dataset도 사용한다.

- **Office-Caltech10**: Amazon, Caltech, DSLR, Webcam 등 서로 다른 촬영 환경과 카메라 source.
- **DomainNet**: Clipart, Infograph, Painting, Quickdraw, Real, Sketch 등 서로 다른 이미지 스타일.
- **ABIDE**: 서로 다른 의료기관에서 수집된 fMRI data. 기관별 장비와 프로토콜 차이가 feature shift를 만드는 의료 FL 예시다.

여기서 각 source 또는 기관을 client로 두어 client마다 feature distribution이 다른 상황을 만든다. 이 모든 feature shift non-IID 설정에서 FedBN이 FedAvg와 FedProx를 능가한다.

## Takeaways

FedBN은 FedAvg의 aggregation 대상을 조정하여, feature shift가 강한 non-IID FL에서 BN layer를 client-specific adaptation module로 활용한 간단하고 강력한 방법이다. BN을 FL에 적극적으로 도입한 대표적인 초기 연구이기도 하다.

다만 결과를 읽을 때는 몇 가지를 구분해서 볼 필요가 있다. 논문이 한 것과 하지 않은 것을 정리하면 다음과 같다.

**한 것**

- feature shift를 명시적으로 정의하고, Assumption 4.1로 이론적 상황을 수학화했다.
- toy example로 BN이 loss surface를 harmonize할 수 있음을 보였다.
- benchmark에서 client별 domain을 다르게 구성하고, real-world multi-domain/multi-site dataset에서 성능을 검증했다.

**하지 않은 것**

- 모든 real-world dataset에 대해 $P_i(x)$, $P_i(x \mid y)$를 엄밀히 추정해 feature shift를 통계적으로 증명한 것은 아니다.
- 각 client 간 feature distribution distance를 정량화해 "이만큼 feature shift가 있다"고 직접 측정한 것은 제한적이다.
- 이론 증명은 일반적인 CNN 전체에 대한 완전한 증명이 아니라, 단순화된 over-parameterized two-layer ReLU setting에 한정된다.

이러한 한계에도 불구하고, "무엇을 평균내지 않을 것인가"라는 최소한의 개입만으로 feature shift 문제에 대응한다는 점에서 FedBN은 이후 personalized FL 연구가 참고하는 간결한 baseline이 되었다.
